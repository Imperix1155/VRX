# src/renderer — React UI

## Purpose
The renderer process: the React + Tailwind v4 UI. Runs sandboxed; reaches the main process only through the preload bridge.

## Ownership
- `index.html`, `src/main.tsx` — React entry (`main.tsx` also initializes i18next).
- `src/components/ErrorBoundary.tsx` — React error boundary (VRX-127). Wraps `<App/>` in `main.tsx` (inside `QueryClientProvider`) AND each panel (e.g. `FriendsList` in `App.tsx`). Catches render-phase errors from its subtree and shows a glass-styled fallback (`--error` token + non-color glyph + Reload + Copy-diagnostics, all copy i18n'd). A `variant` prop (VRX-165) controls the fallback: `'app'` (default) is full-screen with the brand mark; `'panel'` is compact (no `min-h-screen`, no brand mark) so a single panel's error doesn't blank the whole window — the per-panel boundary passes `variant="panel"`. `componentDidCatch` logs to `electron-log/renderer`, which forwards to the main process via the IPC bridge wired by `initLogger()` — no new IPC channels needed. Tested via `@testing-library/react` + `// @vitest-environment jsdom` (the jsdom environment is required because React error boundaries only fire during the client commit phase). `@testing-library/react` was added to devDependencies for this purpose; `.tsx` component tests that need a real DOM use the `// @vitest-environment jsdom` header to opt in. Audit W6 added: `LoginScreen.test.tsx` (full credentials→2FA→verify flow incl. the password-dropped-on-2FA security pin and bridge-failure surfacing), `TopBar.test.tsx` (onlineCount derivation + plurals; stubs `ResizeObserver`), `utils/loginError.test.ts` (code→key table incl. raw-code collapse), and `queries/auth.test.ts` (bridge guards fail closed).
- `src/App.tsx`, `src/components/` — UI components. `App.tsx` is the **auth gate** (VRX-158): reads `useAuthStatus` and renders `LoginScreen` unless authenticated (blank while the check is pending, so an already-signed-in session doesn't flash the form), otherwise the **§8 `AppShell`**. ## Local Contracts
- Design tokens are the single source of truth (DESIGN.md §2/§2A, defined in `assets/main.css`). NEVER hardcode color/spacing outside tokens.
- Themed colors are raw CSS vars consumed via arbitrary utilities (`bg-[var(--vrc)]`) so they flip under `[data-theme="light"]`; only the static scale (radius/fonts) lives in `@theme`.
- Tailwind v4 drops opacity modifiers on arbitrary vars (`bg-[var(--x)]/N` → solid) — use `color-mix()` or theme colors for tints.
- Dark is the default; light is a `[data-theme="light"]` override (parity, not a fork).
- Honor `prefers-reduced-motion` via Tailwind `motion-safe:`. No `!important`.
- Sandboxed: no node/electron access; reach main only through the preload-exposed API.
- User-facing strings go through i18next (`useTranslation`/`t('key')`) — never hardcode copy. Add keys to `locales/en/translation.json` (and peer locales). Resources are bundled (synchronous, no Suspense); initial language is the OS locale (`navigator.language`), fallback English.
