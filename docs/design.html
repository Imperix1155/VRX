VRX

The VRX visual language

VRX is a dark, translucent liquid-glass companion for VRChat & ChilloutVR. Frosted panels float over a slow aurora the glass refracts; a faint CRT scanline texture and a VT323 retro-terminal accent give it an "old-internet" soul that sets it apart from both platforms' own apps. It is unique, but operational and readable first. Color always carries meaning through a fixed channel system — never decoration. This guide shows every element: how it looks, how it's built, and the choice behind it.

● Liquid Glass — locked baseline

Why these choices
Blue=VRChat and orange=ChilloutVR is the whole brand: two ecosystems merging, with a neutral silver bridge (the mark's "R", "hot" stat — slated to become a user-customizable accent later). Values are UI-tuned (a touch desaturated) so they stay legible on dark glass — deeper hero hues live separately and never appear in app chrome.

03 · Material
The glass surface

The single most important element. Everything that "floats" — sidebar, cards, the segmented control, list containers — is built from one .glass recipe.

3.1 Base glass panel
Translucent white fill + heavy backdrop blur + a 1px light border + a depth shadow stacked with an inset top highlight, plus a top-left specular sheen.
content sits here
How it's built
.glass { background: linear-gradient(135deg, rgba(255,255,255,.09), rgba(255,255,255,.025)); backdrop-filter: blur(26px) saturate(165%); border: 1px solid rgba(255,255,255,.13); border-radius: 20px; box-shadow: 0 12px 44px rgba(0,0,0,.5), /* depth */ inset 0 1px 0 rgba(255,255,255,.22); /* edge */ } .glass::before { /* top-left sheen */ background: radial-gradient(125% 80% at 0% 0%, rgba(255,255,255,.11), transparent 46%); }
Why
The saturate(165%) is what makes it read as "liquid glass" rather than a gray frost — it super-charges the aurora bleeding through.
The inset top highlight is non-negotiable: paired with the depth shadow it gives the lit top edge / shadowed underside that sells a physical pane of glass. Without it the panel looks flat.
The ::before sheen is the wet specular glint — like light catching the corner of a real screen.
Light mode keeps every one of these moves — frosted pane, border, inset highlight, depth shadow, sheen all remain required — but inverts the fill: a bright white pane (≈0.70→0.28 white) with ink borders/shadow instead of translucent-white-on-black. Same material, lit. See §11.
3.2 Platform-tinted glass
On surfaces that belong to one platform (hot-instance cards), the glass itself is tinted — so the material carries platform identity.
.tint-vrc
.tint-cvr
How
.tint-vrc { background: linear-gradient(135deg, rgba(43,124,232,.22), rgba(43,124,232,.05)); border-color: rgba(43,124,232,.34); } .tint-cvr { /* same, orange 243,113,30 */ }
Why
Tint opacity is capped around 0.22 — push past that and the frost reads as solid colored plastic, losing the glass. Radius scale: 20px panels/cards, 12–13px nav & segmented control & buttons, 9–10px small pills/affordances.
04 · Atmosphere
The background the glass refracts

Look behind this very page: a near-black canvas with a quiet blue glow (top-left) and orange glow (bottom-right) — the two platforms as light — plus a faint CRT scanline grid. The glass has something worth blurring.

How
body::before { /* aurora, z-index:-2 */ background: radial-gradient(58% 50% at 12% 6%, vrc .26, …), radial-gradient(58% 50% at 92% 96%, cvr .20, …), var(--bg-base); /* near-black #08080b */ } body::after { /* scanlines, z-index:-1 */ background: repeating-linear-gradient(0deg, rgba(255,255,255,.022) 0 1px, transparent 1px 3px); mix-blend-mode: overlay; opacity: .6; }
Why
The blue (VRChat) corner and orange (ChilloutVR) corner are the brand, expressed as light — the only color on an otherwise neutral near-black canvas. Two platforms glowing at opposite corners, no third hue: neutral, premium, and unmistakably VRX.
The scanlines are the old-internet soul — at 2.2% white on overlay they're a texture you feel more than see. Keep the aurora slow/static in v1; if it ever animates it must respect prefers-reduced-motion.
Light mode keeps the same composition — same blue-top-left / orange-bottom-right corners, same scanlines — but over a near-white canvas, with the glows softened (blue .18 / orange .15) and the scanlines flipped to dark ink on multiply (white-overlay scanlines would vanish / look like dirt on white). "Sunlit, not rebranded." See §11.

5.2 State — the avatar dot
System-assigned presence — where a friend is, not their chosen status. Three values, shown only as the dot on their avatar.
In a world · green
Active — online, not in game · teal
Offline · gray
How
.st-ingame { background: var(--ingame); } /* green */ .st-active { background: var(--active); } /* teal */ .st-offline { background: var(--offline); }/* gray */
Why
In-game (state:"online") and offline are the shared baseline — same meaning on both platforms. Active (state:"active") is VRChat-only: online on the website or desktop app but not in a world. Teal (not blue) keeps the dot clear of the platform channel; it only ever appears as the dot.
5.3 VRChat status — a labeled pill
VRChat lets a user broadcast how they want to be approached. Four statuses, each with a canonical color — shown as a small labeled pill so the color is unambiguous (it never appears as a bare dot fighting the platform spine or the state dot).
Join Me Online Ask Me DND ✎ + custom status · up to 32 chars
Data model — VRChat API (two fields!)
status: "join me" | "active" | "ask me" | "busy" // "active" → Online (green); "busy" → DND (red) statusDescription: "custom text" // ≤32 chars state: "online" // in-game (the dot, 5.2) | "active" // online, NOT in game | "offline"
Behavior & why
Join Me auto-accepts join requests · Online is joinable if the instance allows · Ask Me & DND hide the world (so we show the status, not the instance) · DND also silences requests.
ChilloutVR has none of this — only online/offline. We don't fake a CVR equivalent; CVR rows simply have no status pill.
5.4 Joinability — the derived action
The status pill (5.3) says how a friend wants to be approached; the right-side affordance is the action you take, derived from status + instance: join freely, request an invite, or — for Do Not Disturb — don't.
Join Ask ⊘ DND + active / offline → no action
How
.aff.join2 { /* glass-filled, primary */ } .aff.ask { border: 1px solid rgba(255,255,255,.16); } .aff.busy { color: var(--text-faint); }
Why
Status colors live in the labeled pill (5.3); the action stays neutral so the same blue/orange/red never reappears as a second, unlabeled signal. The affordance maps the status to what it means for you: Join Me & Online → Join, Ask Me → Ask, DND → ⊘.
5.5 Trust — demoted by design
VRChat itself is phasing out trust ranks (an in-game "show as User" toggle; nametags no longer rank-colored). VRX mirrors that.
Show trust ranks — off by default
Nova V (default — name neutral)
Nova Known (opt-in pill, muted)
CVR has no trust ranks at all — we don't invent a fake equivalent. That asymmetry is honest. Names stay neutral so trust never becomes a 5th color competing with the four above.

07 · Type
Two typefaces, one rule

Inter does all the reading. VT323 is an accent voice — never body text.

7.1 Inter — everything you read
Regular 400 Medium 500 Semibold 600 Bold 700 Extrabold 800
Names, labels, descriptions, buttons, world titles, statuses — anything the user scans or reads repeatedly.
7.2 VT323 — the accent voice
VRX 47 Hot Instances V
Where VT323 is allowed
The VRX mark · big stat numbers · section kickers · the V/C glyphs · technical IDs/versions. That's the whole list.
Why the strict limit
VT323 is a CRT terminal face — gorgeous as a flavor, punishing as body text. Confining it to numerals and short all-caps labels gives the retro character without ever hurting readability. Never set descriptions, helper text, statuses, or form labels in it.
08 · Layout
The app shell

A fixed 248px glass sidebar and a single scrolling content column. The shell never scrolls; only the main column does.

VRX
VRX · v0.1.0
How
.app { display: grid; grid-template-columns: 248px 1fr; height: 100vh; padding: 16px; gap: 16px; } body { overflow: hidden; } /* shell is fixed */ .main { overflow-y: auto; } /* only this scrolls */
Why
A persistent, calm sidebar (brand → nav → footer) and a dense content column is the shape of a desktop utility, not a landing page. The active nav item gets a glass fill and a left spine that runs blue→orange — the merge again. v1 is desktop-only; no responsive collapse required.

10 · The biggest problem
Cross-platform friend linking

A friend can exist on both platforms under different names, with no shared identity between them. So linking is user-driven — the user says "this VRChat friend is that CVR friend," and VRX stores the relation. We never auto-merge.

R
Rin
linked across 2 platforms
Vin The Great Pug · PublicJoin
COffline · 2h ago
Why user-driven
There is no reliable signal that two accounts are the same person. A wrong auto-merge is worse than no link, so the user always confirms. VRX may suggest matches (e.g. identical display names), never apply them silently.
How it renders
One person card, blue→orange avatar (the bridge), each platform's presence shown independently — Rin can be in-world on VRChat and offline on CVR at once. Link data is private, stored in the app's own userData — never written into VRCX/CVRX folders.
11 · Light mode
Same app, lights on

Dark is the default and the focus. Light mode is fully supported — but it is not a second design. It is the exact same VRX: same layout, same components, same channel law (§05), same openness ladder (§06), same typography (§07), same glyphs. Only the tokens, glass material, and background shift lighter, behind a single [data-theme="light"] selector on <html>. A user who flips the switch should feel like someone turned the lights on — not like they opened a different product. (See the live dark/light pair in §01.)

11.1 What changes vs. what stays
The discipline that keeps the two themes in parity: a short list changes, everything structural stays.
Shifts in light
• Tokens — base, text, platform, state, status all retune (below)
• Glass material — white pane instead of translucent-white-on-black
• Background — near-white canvas; glows softened; scanlines → dark-ink multiply
• Contrast tuning — borders/shadows become ink; pill text darkens
Stays identical (theme-invariant)
• Layout & app shell (§08) — pixel-for-pixel
• The channel law (§05) — platform/state/status/openness/trust, same locations
• Openness ladder (§06) — same icons, same neutral badges
• Typography (§07) — Inter + VT323, same roles
• Every component (§09) — no light-only classes, ever

11.3 Light glass + background — the material, lit
A live light-themed panel, rendered right here. Every glass "move" from §03 survives — pane, border, inset highlight, depth shadow, sheen — just inverted to bright.
.tint-vrc
.tint-cvr
How (override layer)
/* dark is default; light is an override */ [data-theme="light"] .glass { background: linear-gradient(135deg, rgba(255,255,255,.70), rgba(255,255,255,.28)); border-color: rgba(40,48,68,.16); box-shadow: 0 16px 46px rgba(59,72,93,.22), inset 0 1px 0 rgba(255,255,255,.88); } [data-theme="light"] body::before { /* same corners, softer */ background: radial-gradient(58% 50% at 12% 6%, vrc .18, …), radial-gradient(58% 50% at 92% 96%, cvr .15, …), linear-gradient(180deg,#fbfcff,var(--bg-base),#e7edf4); }
Why
Tint ceiling drops to 0.18 in light (vs 0.22 dark) — higher and the platform color overpowers the white pane and the app stops feeling like glass.
Scanlines flip to dark ink on multiply at low opacity — a white overlay would be invisible on a light canvas. Same texture, opposite ink.
Openness badges stay neutral (white/ink-gray) — never blue/orange — exactly as in dark.

12 · Guardrails
Do & Don't

Do

  • Build every floating surface from .glass
  • Pull all color/spacing from the tokens
  • Say platform with tint + spine + glyph
  • Keep openness badges neutral gray + shared icon
  • Pair every color with position + glyph/text
  • Honor prefers-reduced-motion
  • Use official platform logos, object-fit:contain
  • Keep dark + light in parity: light is overrides only (§11)

Don't

  • Color an openness badge by platform or type
  • Use blue/orange for anything but platform
  • Make "online" blue (it's teal)
  • Turn trust ranks on by default, or color names
  • Set body/helper/status text in VT323
  • Auto-merge cross-platform identities
  • Invent CVR features (trust, Busy) for symmetry
  • Fake a VRChat/ChilloutVR logo
  • Fork components for light mode — retune tokens, never duplicate classes
Full token values & CSS (both themes) live in glass.html · light overrides in DESIGN.md §2A–§4A · machine-readable rules for AI agents live in DESIGN.md.
